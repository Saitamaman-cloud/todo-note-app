# 今日メモTodo

## v11: 生活ルーチン管理

既存の ToDo / メモ / カレンダーに加えて、家事や生活作業を繰り返し管理する「ルーチン」画面を追加しました。

- 下部ナビゲーションから「ルーチン」を開けます。
- 掃除、洗濯、ゴミ、買い物、ペット、健康、手続きなどのカテゴリを使えます。
- 繰り返しは毎日、N日ごと、毎週、毎月、手動に対応しています。
- 次回予定日、予定時刻、最終実施日、有効/無効を管理できます。
- 今日までにやるルーチンをホームとルーチン画面に表示します。
- 「実施済み」で最終実施日と次回予定日を更新し、実施履歴を残します。
- ルーチンから今日の ToDo へ追加できます。
- バックアップには `routines` と `routineHistories` も含まれます。
- 古いバックアップにルーチン情報がなくても復元できます。

IndexedDB は既存データを残したままバージョン 2 に更新され、`routines` と `routineHistories` ストアが追加されます。

## 生活ルーチン管理 詳細仕様

### 追加した画面

下部ナビゲーションに「ルーチン」を追加しました。ルーチン画面では、今日までに実施する必要があるルーチン、フィルタ付きのルーチン一覧、新規作成、編集、実施済み、ToDoへの追加を操作できます。

ホーム画面にも「今日やるルーチン」を表示します。`nextDueDate` が今日以前で、かつ `isActive` が `true` のルーチンが対象です。

### ルーチンのデータ構造

`routines` ストアには以下の項目を保存します。

- `id`: ルーチンの一意ID
- `title`: ルーチン名
- `category`: 掃除、洗濯、ゴミ、買い物、ペット、健康、手続き、家計、車、その他などの分類
- `description`: 補足メモ
- `repeatType`: 繰り返し種別
- `intervalDays`: N日ごとの間隔
- `daysOfWeek`: 毎週指定の曜日
- `dayOfMonth`: 毎月指定の日
- `lastDoneDate`: 最終実施日
- `nextDueDate`: 次回予定日
- `defaultTime`: ToDoへ追加するときの予定時刻
- `isActive`: 有効/無効
- `autoCreateTodo`: 保存時に予定日のToDoへ追加する設定
- `createdAt`: 作成日時
- `updatedAt`: 更新日時

### 実施履歴のデータ構造

`routineHistories` ストアには以下の項目を保存します。

- `id`: 履歴の一意ID
- `routineId`: 対象ルーチンID
- `doneDate`: 実施日
- `note`: 履歴メモ
- `createdAt`: 作成日時

ルーチンを「実施済み」にすると、`lastDoneDate` を今日の日付に更新し、`routineHistories` に履歴を追加します。

### 繰り返し種別

対応している繰り返しは以下です。

- 毎日: 実施日の翌日を次回予定日にします。
- N日ごと: 実施日から `intervalDays` 日後を次回予定日にします。
- 毎週: 実施日より後で、`daysOfWeek` に含まれる最も近い曜日を次回予定日にします。
- 毎月: 翌月以降の `dayOfMonth` を次回予定日にします。指定日が存在しない月は、その月の末日を使います。
- 手動: 次回予定日は自動計算しません。編集画面で手動変更します。

期限超過のルーチンを実施済みにした場合でも、実施日は今日として扱い、今日を基準に次回予定日を計算します。

### 一覧とフィルタ

ルーチン一覧は次回予定日が近い順に並びます。並び順は、期限超過、今日、今後の予定、予定日なし、無効ルーチンの順です。

フィルタは以下に対応しています。

- 今日やる
- 期限超過
- 有効
- 無効
- すべて
- カテゴリ別

### ToDoへの追加

ルーチンごとに「ToDoへ追加」を押すと、ルーチンの予定日に合わせてToDoを追加します。`nextDueDate` が今日以降ならその日付へ、期限超過なら今日の日付へ追加します。ToDoのタイトルは `routine.title`、時刻は `routine.defaultTime`、ステータスは `todo` です。

ルーチン編集画面で「保存時に予定日のToDoへ追加する」をONにして保存すると、保存直後にToDoが作成されます。繰り返しルーチンの場合は、次回予定日から90日先までの予定をToDoへ展開します。たとえば毎週月曜日なら、最初の月曜日だけでなく、その後の月曜日分も重複を避けて作成されます。追加後は最初の作成先日付を選択したToDo画面へ移動するため、保存結果をそのまま確認できます。

「すべてToDoへ」では、今日やるルーチンをまとめて今日のToDoへ追加します。同じ日の同じタイトルのToDoがすでにある場合は重複としてスキップします。個別追加では、重複時に確認ダイアログを表示します。

### バックアップと復元

バックアップJSONには、既存の `todos` と `notes` に加えて、`routines` と `routineHistories` を含めます。

復元時は、古いバックアップに `routines` や `routineHistories` が存在しない場合でもエラーにせず、空配列として扱います。これにより、ルーチン機能追加前のバックアップも復元できます。

### PWAキャッシュ

Service Worker のキャッシュ名を `today-memo-todo-cache-v12` に更新しました。読み込み対象も `style-v11.css?v=12`、`db-v11.js?v=12`、`app-v11.js?v=12` に更新しています。

個人用の「ToDo + 複数ページメモ」PWAです。スマホで今日のToDoを追加し、状態を変え、自由メモを複数ページ保存できます。データはログインなしで端末内のIndexedDBに保存します。

## ローカル起動方法

1. ターミナルで `todo-note-app` フォルダへ移動します。
2. 次のコマンドを実行します。

```bash
python -m http.server 8000
```

3. ブラウザで `http://localhost:8000/` を開きます。

PWAとService Workerは `file://` では正しく動かないため、ローカルサーバー経由で確認してください。

## スマホで使う方法

同じWi-Fi内のスマホからPCのIPアドレスを使って開きます。

```text
http://PCのIPアドレス:8000/
```

Android Chromeでの利用を主対象にしています。iPhone Safariでもホーム画面追加と基本利用ができる構成です。

## ホーム画面に追加する方法

### Android Chrome

1. Chromeでアプリを開きます。
2. メニューから「ホーム画面に追加」または「アプリをインストール」を選びます。
3. 追加後、ホーム画面のアイコンから起動します。

### iPhone Safari

1. Safariでアプリを開きます。
2. 共有ボタンを押します。
3. 「ホーム画面に追加」を選びます。

## バックアップ方法

設定画面の「バックアップ出力」を押すと、ToDoとメモの全データをJSONファイルとして保存します。

ファイル名は次の形式です。

```text
today-memo-todo-backup-YYYYMMDD.json
```

## 復元方法

1. 設定画面の「バックアップ復元」を押します。
2. 保存済みのJSONバックアップファイルを選びます。
3. 確認ダイアログで同意すると、既存データを全削除してバックアップ内容に置き換えます。

## 注意事項

- データは使用中のブラウザ内に保存されます。
- ブラウザデータやサイトデータを削除すると、保存内容が消える可能性があります。
- クラウド同期、ログイン、画像添付、通知、共有機能は初期版にはありません。
- 大切なメモは定期的にバックアップしてください。
